Суть приложения - пользователь должен иметь возможность:
• Авторизоваться\зарегистрироваться,
• Публиковать посты, редактировать посты, удалять посты, получать посты, должна
быть возможность комментировать посты другим пользователем.


Сделать приложение с 4 микросервисами:
1. Auth Api – система авторизации, должны быть методы
• Регистрации
• Авторизации
• Схема авторизации Bearer Token по логину и паролю


2. Blog Api – публикация постов. Посты должны быть простые (заголовок и тело). Сделать на БД Postgres


3. Chat Api – мини чат с хранилищем кэша Redis и SignalR


4. Weaher Api – сделать интеграцию с https://openweathermap.org/api по получению погоды по гео-точкам. Хранить все запросы и результаты в БД Mongo

Все сервисы должны общаться между собой через rabbit rpc (или аналоги).
